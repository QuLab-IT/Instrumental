PHASe:UNLock:ERRor:STATe {OFF|ON}
PHASe:UNLock:ERRor:STATe?

Disable or enable the function generator from generating an error if the phase-lock is ever lost. The default is OFF. If the phase-lock is lost and the error is enabled, a "Reference phase-locked loop is unlocked" error is generated. The unlock error setting is stored in non-volatile memory. The :STAT? query returns "0" (OFF) or "1" (ON).
